## 核心发现：RAG 实体覆盖率低的真相

### 问题
报告中 `entity_coverage` 普遍很低（0-20%），但综合评分却有 8.92/10

### 根本原因
**实体类型不匹配**：
- RAG 检索实体：宏观历史（"恢复高考"、"邓小平南巡"、"亚洲金融危机"）
- 生成文本实体：个人叙事（"老张"、"德明"、"林老板"、"窑洞"）
- 两者在不同语义层面，重叠率自然低

### 解决方案
在报告中分别列出两类实体：

```json
{
  "entities": {
    "rag_retrieved": ["恢复高考", "邓小平", ...],      // RAG检索到的
    "rag_used_in_text": ["恢复高考"],                  // 在文本中使用的
    "rag_coverage": 0.2,                               // 覆盖率
    "text_extracted": ["老张", "德明", ...],           // 文本中的所有实体
    "text_only_entities": ["老张", "德明", ...]        // 仅在文本中的
  }
}
```

### 核心洞察
- **RAG 覆盖率低（20%）≠ RAG 无效**
- RAG 以背景方式融入，而非罗列实体名
- 好的生成 = 保留原文细节（70-80%）+ 融入关键历史事件（1-2个）
