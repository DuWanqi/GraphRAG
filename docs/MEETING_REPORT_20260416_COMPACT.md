# 回忆录 RAG 系统精简报告

> 日期：2026-04-16  
> 项目：GraphRAG 长文生成与质量控制

---

## 核心逻辑

**输入**：跨越数十年的回忆录原文  
**流程**：时间边界分段 → 逐段检索知识图谱 → 注入跨章上下文生成 → 多维评估 → 质量门控  
**输出**：完整润色长文 + 质量报告

---

## 一、需求-更新逻辑链

### 1.1 核心需求
- **分段需求**：长文按时间边界切分，保证检索精度
- **生成需求**：各章连贯不重复，保留原文细节，融入历史背景
- **质量需求**：可交付判定机制，不达标自动重试

### 1.2 技术更新链

```
需求：时间精准分段
  ↓
实现：优先级切分策略（年份 > 标题 > 空行 > 长度）
  ↓
关键：不跨时间边界合并短段
  ↓
输出：带元数据的段列表（年份/地点/人物）

需求：各章连贯不重复
  ↓
实现：ChapterContext 跨章上下文管理
  ↓
关键：前文摘要 + 反重复要点（jieba分词+停用词过滤）+ 位置指令
  ↓
输出：每章生成时知道"前面写了什么"

需求：可交付判定
  ↓
实现：质量门控 7 维度检查
  ↓
关键：字数/综合分/FActScore/跨章重复/总结句/套话/感悟收尾
  ↓
输出：通过/不通过 + 修复计划 + 自动重试
```

## 二、五步 Pipeline

### STEP 1：分段（memoir_segmenter.py）

**优先级策略**：
1. **最高**：段首新年份（"一九七二年"）→ 强制切分
2. 次高：章节标题（"第一章"）
3. 次低：空行
4. 兜底：长度约束（超长切分，过短合并）

**关键规则**：合并短段时检测时间边界，不跨年代合并
```python
crosses_time = _LEADING_YEAR.match(下一段)
if 当前段太短 and not crosses_time:
    合并  # 200字段可合并
else:
    不合并  # 即使只有200字，因为跨年代
```

**输出**：每段附带 SegmentMeta（年份/地点/人物/切分原因）

---

### STEP 2：字数预算（chapter_budget.py）

**策略**：段原文字数 × 扩展系数（0.8~2.0）  
**目的**：避免均摊导致的长短失衡

---

### STEP 3：逐章生成（long_form_orchestrator.py）

对每章执行 5 个子步骤：

#### 3a. 检索
```python
rr = await retriever.retrieve(段文本, top_k=10)
```
- 用段文本查知识图谱
- 返回实体/社区/结构化上下文
- **优化**：预取下一章检索（与当前章生成并行）

#### 3b. 构建跨章上下文
```python
cross_ctx = chapter_ctx.build_prompt_section(章索引)
```

注入三块内容：
1. **前文概要**（最近3章摘要）
   ```
   - 第1章 (1972-1977): 知青下乡到陕北张家塬...
   - 第2章 (1977-1978): 恢复高考消息传来...
   ```

2. **反重复要点**（前面所有章的高频短语，最多12个）
   ```
   以下要点已在前文出现，请勿再次展开：
   知青、黄土高坡、柴油机、恢复高考、窑洞、煤油灯
   ```
   
   **关键技术**：jieba 分词 + 停用词过滤
   - 旧方案（字符滑窗）：产出"产队""里分""我们""时候"等无意义碎片/通用词
   - 新方案（jieba）：只保留实义词，有效要点率 100%

3. **位置指令**（开篇/中间/结尾差异化）
   - 开篇：自然引入，不要末尾总结
   - 中间：与前文衔接，禁止感悟收尾
   - 末章：可适度收束

#### 3c. 调 LLM 生成
```python
gr = await generator.generate(
    memoir_text=段文本,
    retrieval_result=检索结果,
    length_hint="425-575字",
    chapter_context=跨章上下文
)
```

**Prompt 结构**：
```
[System] 你是回忆录润色作家，保留原文细节，融入历史背景，
        禁止总结句/套话/"那年"开头/感悟收尾

[User]
  一、回忆录原文（必须保留人名/地名/对话/情节）
  二、时间背景（年份/地点）
  三、历史背景信息（知识图谱检索结果）
  四、跨章上下文（前文概要+反重复要点+位置指令）
  五、写作要求（11条硬约束）
```

#### 3d. 重复检测
```python
rep_warning = chapter_ctx.detect_repetition_with_previous(生成文本)
```
- 提取当前章高频短语，与前面章做交集
- 重叠率 ≥ 15% → 自动重试（temperature +0.1）

#### 3e. 记录本章
```python
chapter_ctx.record_chapter(索引, 生成文本, 实体列表)
```
提取三项（纯规则，不调 LLM）：
- 摘要：按句号截取首句（≤60字）
- 年份范围：正则提取 min-max
- 高频短语：jieba 分词 → 停用词过滤 → 统计 top 8

---

### STEP 4：评估（long_form_eval.py）

#### 段级指标（8项）
| 指标 | 计算方式 |
|---|---|
| entity_coverage | 检索实体在生成文本中的覆盖率 |
| time_consistency | 生成文本是否包含参考年份 |
| keyword_overlap | 检索关键词匹配数 |
| semantic_similarity | 原文 vs 生成文本的 2-gram 余弦相似度 |
| length_score | 生成字数是否在预算区间 |
| paragraph_structure | 段落数量是否合理 |
| transition_usage | 是否使用过渡词 |
| descriptive_richness | 是否有感官词/比喻/情感词 |

**综合分** = 8项加权平均 × 10（0-10分）

#### FActScore 事实检查（3步）
```
生成文本
  ↓
Step A: 原子事实分解（按句切分）
  ↓
Step B: 过滤文学修辞（只保留含数字/时间/地名/事实动词的句子）
  ↓
Step C: 逐条验证
  ├─ 快速路径：规则匹配原文（≥50%关键词命中→支持）
  └─ LLM路径：剩余事实批量验证（上下文=原文+检索背景）
  ↓
FActScore = 被支持事实数 / 总事实数
```

#### 跨章指标（3项）
| 指标 | 含义 |
|---|---|
| inter_chapter_repetition | 相邻章 6-gram 重叠率（评分 = 1 - 重叠率） |
| style_consistency | 各章平均句长变异系数（CV≤0.2→满分） |
| summary_sentence_ratio | 含"总之/综上"等词的句子占比 |

**并行化**：各章评估用 `asyncio.gather` 并发执行

---

### STEP 5：质量门控（quality_gate.py）

#### 逐章检查（7维度）

| 维度 | 检测方式 | 阈值 | 级别 |
|---|---|---|---|
| 字数 | 实际/目标比率 | <0.4 或 >2.5 | error/warning |
| 综合分 | STEP 4 段级评分 | <5.0 | error |
| FActScore | STEP 4 事实支持率 | <60% | error |
| 总结句 | 正则匹配15个模式 | 占比>30% | warning |
| 套话 | 正则匹配8个模板语 | ≥2处 | warning |
| 感悟收尾 | 末尾3句匹配7类模式（仅非末章） | 命中即触发 | warning |
| 跨章重复 | 相邻章 6-gram 重叠率 | >20% | error |

**判定逻辑**：
- 任一 error → `passed = False`
- 自动生成 `RemediationPlan`（哪几章重做/原因/修复建议）

#### 自动重试闭环
```python
while not passed and retry < 2:
    重新检索 + 构建上下文 + 追加修复指令
    重新生成（temperature +0.05）
    替换旧结果
    重新评估
```

## 三、关键技术实现

### 3.1 反重复要点提取：jieba 分词方案

**原理**：
```
原文 → jieba分词 → 过滤（≥2字中文词 + 排除停用词）
     → 统计词级1-gram和2-gram → 取频次≥2的top 8
```

**三层过滤**：
1. jieba 分词：按词边界切分，避免跨词碎片
2. 正则 `[\u4e00-\u9fa5]{2,}`：过滤单字词/标点/数字
3. 停用词表（80+条）：过滤代词/虚词/泛化时间词

**效果对比**（同一段文本）：

| 方案 | 提取结果 | 有效要点率 |
|---|---|---|
| 字符滑窗 | **我们**, 生产队, 父亲, **时候**, 窑洞, **生产**, **产队**, **那时候** | 37.5% |
| jieba分词 | 父亲, 窑洞, 老张, 生产队, 柴油机, 县城, 张家, 黄土 | 100% |

### 3.2 FActScore 双证据源

**问题**：只用检索背景验证，原文中的人名/对话被判"无据"  
**方案**：验证上下文 = **回忆录原文 + 检索背景**  
**效果**：FActScore 从 ~45% 提升到 ~70%

### 3.3 检索预取并行化

```python
# 生成第 k 章时，预取第 k+1 章检索
if i + 1 < len(segments):
    next_retrieval = asyncio.create_task(
        retriever.retrieve(segments[i+1].text, ...)
    )
```
**效果**：检索延迟被 LLM 生成时间隐藏

## 四、实例说明

### 输入样本
`long_memoir_sample.txt`：约 3500 字回忆录，跨越 1972-2008 年，包含：
- 1972 年陕北知青生活
- 1977 年恢复高考
- 1988 年深圳创业
- 1992 年南巡后自主创业
- 1998 年金融危机
- 2008 年回乡

### 分段结果
6 个段落，每段带元数据：
```
段0: 1972-1972, 地点[陕北,张家塬], 人物[老张,赵德明]
段1: 1977-1978, 地点[陕北,北京,武汉], 人物[老张队长,赵德明]
段2: 1988-1988, 地点[北京,深圳,华强北], 人物[林老板]
段3: 1992-1995, 地点[深圳,华强北,香港,北京], 人物[邓小平]
段4: 1998-1999, 地点[深圳,北京], 人物[联想,海尔]
段5: 2008-2008, 地点[北京,陕北,张家塬,武汉,深圳], 人物[张小军,德明]
```

### 生成结果
`long_form_e2e_output.txt`：6 章共 4081 字，每章：
- 保留原文人物/对话/细节
- 融入历史背景（知识图谱检索）
- 无跨章重复（6-gram 重叠率 0.2%）
- 无总结句（0/107 句）

### 实体分析（RAG vs 文本）

#### 第 1 章（1977年恢复高考）

**RAG 检索到的实体 (5 个)**：
- 恢复高考、邓小平、十一届三中全会、知青返城、高等教育改革

**RAG 实体使用情况 (1 个, 覆盖率 20%)**：
- ✅ 恢复高考（直接出现在文本中）

**生成文本中的实体 (9 个)**：
- 老张、张队长、赵德明（原文人物）
- 北京、张家塬、武汉（地点）
- 公社、生产队、知青（组织/群体）

**分析**：
- RAG 覆盖率低（20%），但历史背景以氛围方式融入
- 生成文本保留了原文的人物和地点
- 历史事件"恢复高考"作为时代标记明确提及

#### 第 4 章（1992年南巡）

**RAG 检索到的实体 (5 个)**：
- 邓小平南巡、南方谈话、市场经济、深圳特区、改革开放

**RAG 实体使用情况 (2 个, 覆盖率 40%)**：
- ✅ 邓小平南巡（直接提及）
- ✅ 深圳特区（间接提及"经济特区"）

**生成文本中的实体 (10 个)**：
- 老刘、老陈、林老板、女朋友（原文人物）
- 邓小平（历史人物）
- 北京、深圳、华强北、香港（地点）
- 研究所（机构）

**分析**：
- RAG 覆盖率较高（40%），历史事件明确提及
- 同时保留了原文的个人叙事
- 历史与个人叙事结合良好

**核心发现**：
- RAG 实体（宏观历史）vs 文本实体（个人叙事）在不同语义层面
- RAG 覆盖率低（20%）≠ RAG 无效，而是以背景方式融入
- 好的生成 = 保留原文细节（70-80%）+ 融入关键历史事件（1-2个）

## 五、先前问题及修复

| 问题 | 修复措施 | 对应代码 |
|---|---|---|
| 分段不按年代切 | 不跨时间边界合并短段 | `memoir_segmenter.py` |
| 各章重复/套话 | ChapterContext + jieba分词反重复 + prompt禁止 | `chapter_context.py`, `prompts.json` |
| 无可交付判定 | 质量门控 + 修复计划 + 自动重试 | `quality_gate.py` |

| 生成千篇一律 | prompt改为"润色改写" + 字数预算按段扩展 | `prompts.json`, `chapter_budget.py` |
| 章末感悟泛滥 | 位置指令 + 感悟检测正则 | `chapter_context.py`, `quality_gate.py` |

## 七、一页纸总结

**核心流程**：时间分段 → 逐章（检索→跨章上下文→生成→重复检测→记录）→ 评估 → 门控 → 重试

**关键创新**：
1. **时间边界保护**：不跨年代合并短段，保证检索精度
2. **跨章上下文**：前文摘要 + jieba分词反重复要点 + 位置指令
3. **质量门控**：7维度检查 + 自动修复计划 + 重试闭环
4. **双证据源**：原文+背景验证事实，FActScore 提升 25%
5. **并行优化**：评估并行 + 检索预取

**效果**：
- 分段精度：100% 按时间边界切分
- 跨章重复：0.2%（6-gram 重叠率）
- 总结句：0%
- 综合评分：8.9/10
- 可交付判定：自动化质量门控 + 最多2轮重试
